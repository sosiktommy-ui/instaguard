# cookie-gen — локальный генератор сессий Instagram

Мини-инструмент для «упрямых» аккаунтов, которые в облаке циклят на checkpoint/капче.
Логинит аккаунты в **видимом окне браузера на вашем ПК** (Instagram не палит его как бота),
а checkpoint/капчу/2FA-подтверждение вы решаете **глазами**. На выходе — сессия
(`storageState`: cookies + localStorage), которую вставляете в приложение.

Это тот же артефакт, что делает облачный вход (`InstagramAccount.browserState`) —
полностью взаимозаменяемо. См. `plan.md` §2.1.

## Как пользоваться

1. Установить (один раз):
   ```
   cd tools/cookie-gen
   npm install
   ```
   (`npm install` сам докачает Chromium через `playwright install`.)

2. Создать `accounts.txt` рядом со скриптом (скопировать из `accounts.example.txt`):
   ```
   логин пароль
   логин пароль 2fa-ключ
   логин пароль 2fa-ключ прокси(host:port:user:pass)
   ```
   ⚠️ Логиньтесь **через прокси в стране аккаунта** — сессия будет чистой, и после
   импорта действия пойдут через тот же прокси без прыжка IP.

3. Запустить:
   ```
   npm start
   ```
   Откроется окно на каждый аккаунт по очереди. Если Instagram просит код/капчу/подтверждение —
   сделайте это **прямо в окне**. Скрипт ждёт вход до 4 минут, затем сохраняет сессию.

4. Забрать результат:
   - `sessions.txt` — по строке на аккаунт (одна строка = один `storageState`).
   - `output/<логин>.json` — то же по-файлово.

5. Вставить в приложение:
   - **Массово:** «Аккаунты → Импорт списком» → режим «🍪 Куки» → вставить строки из `sessions.txt`.
   - **По одному:** «+ Аккаунт → 🍪 Куки» → вставить JSON из `output/<логин>.json`.

## Заметки
- `accounts.txt`, `sessions.txt`, `output/` — в `.gitignore` (личные данные, не коммитятся).
- Инструмент НЕ обязателен: основной путь — облачный вход в приложении. Это аварийный
  разблокировщик для аккаунтов, которые упорно не входят в облаке.
