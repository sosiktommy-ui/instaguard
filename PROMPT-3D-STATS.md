> Скопируй блок ниже целиком. Он самодостаточен: генерирует красивую интерактивную 3D-вкладку

## 📋 ПРОМПТ (копировать отсюда)

Построй **одну самодостаточную интерактивную вкладку статистики** — single-file React-компонент
(или чистый HTML+CSS+JS, если так проще), который выглядит как премиальная 3D-аналитика уровня
дорогого SaaS-дашборда. Никаких внешних CDN/сетевых запросов — всё инлайн, работает офлайн.

### Цель вкладки
Показать **статистику автоматизации Instagram** с ГЛАВНОЙ метрикой:
**сравнение по типам кампаний — какой триггер срабатывает чаще всего.**
Типы кампаний (ровно эти 4, с этими подписями и цветами):

| Тип (ключ)      | Подпись (RU)        | Цвет      |
|-----------------|---------------------|-----------|
| `NEW_FOLLOWER`  | Новая подписка      | `#7C5CFC` (фиолетовый) |
| `NEW_COMMENT`   | Комментарий         | `#22C55E` (зелёный)    |
| `NEW_LIKE`      | Лайк                | `#F59E0B` (оранжевый)  |
| `STORY_MENTION` | Ответ на сторис     | `#38BDF8` (голубой)    |

### Центральный элемент — интерактивный 3D-график
- **3D pie / donut** с реальным объёмом (боковые грани, тень на «поверхности», лёгкий глянец),
  ровно как «3D Pie Chart — Market Share»: секторы с высотой, мягкие пастельные грани, подпись
  доли % прямо на секторе.
- **Интерактив (всё кликабельно и плавно):**
  - **Drag** мышью — вращение всей фигуры (orbit).
  - **Scroll** — зум.
  - **Клик по сектору** — «взрыв» (explode: сектор выезжает наружу) + выделение в легенде.
  - Кнопки: **Reset View**, **Pause Rotation** (по умолчанию медленное авто-вращение),
    **Explode All**, **Donut Mode** (переключение pie ⇄ donut).
  - **Hover** по сектору — подсветка + всплывающая подсказка (тип, срабатываний, доля %).
- Счётчик **FPS** в углу (маленький, ненавязчивый).
- Реализация 3D — на **чистом Canvas 2D с ручной 3D-проекцией** ИЛИ на WebGL/three только если
  инлайнится без CDN. Приоритет — плавность (60+ FPS) и отсутствие внешних зависимостей.

### Правая панель «BREAKDOWN»
Стеклянная карточка со списком: цветная точка • подпись типа • доля %. Синхронна с графиком
(hover/выделение подсвечивает и там).

### Нижние табы-срезы (переключают набор данных графика)
`Срабатывания по типу` (главный) · `По действию` (DM / Лайк / Подписка / Сторис) ·
`По аккаунту` · `По времени (7 дней)`. Активный таб подсвечен. Переход между срезами —
плавная морф-анимация секторов.

### Верхняя полоса метрик (KPI-карточки, объёмные, с анимированным счётчиком)
Всего срабатываний · Активных кампаний · Выполнено действий · Конверсия %.

### Данные (демо — вставь как редактируемую константу вверху файла)
```js
const CAMPAIGN_STATS = [
  { key: 'NEW_FOLLOWER',  label: 'Новая подписка',  fires: 12, color: '#7C5CFC' },
  { key: 'NEW_COMMENT',   label: 'Комментарий',     fires: 0,  color: '#22C55E' },
  { key: 'NEW_LIKE',      label: 'Лайк',            fires: 0,  color: '#F59E0B' },
  { key: 'STORY_MENTION', label: 'Ответ на сторис', fires: 0,  color: '#38BDF8' },
];
```
**Важно:** график должен корректно работать, когда у части типов `fires: 0` (как в примере: один
тип 12, остальные 0) — покажи это осмысленно (не рисуй нулевые секторы как «дырки»; либо тонкая
метка «0», либо аккуратно исключай нулевые из круга, оставляя их в легенде серыми). Также добавь
демо-данные для остальных срезов (по действию/аккаунту/времени), чтобы табы были живыми.

### Дизайн-система
- **Тёмная И светлая тема** (`prefers-color-scheme` + возможность переключателя); контраст ≥ WCAG AA.
- Стекломорфизм (полупрозрачные карточки + `backdrop-blur`), мягкие тени, скруглённые углы,
  аккуратная типографика, тонкие разделители. Палитра — фиолетовый бренд `#7C5CFC` + акценты выше.
- Всё **адаптивно** (мобайл/планшет/десктоп): на узком экране легенда уходит вниз, график
  ужимается, табы скроллятся горизонтально. Горизонтального скролла body быть не должно.
- Уважать **`prefers-reduced-motion`**: выключить авто-вращение и тяжёлые переходы (график
  остаётся статичным, но интерактив по клику работает).
- Плавные микро-анимации (появление карточек, счётчики, морфинг секторов). Ничего мигающего/резкого.

### Технические требования
- **Один файл**, без внешних запросов (шрифты — системные; иконки — инлайн SVG или юникод).
- Чистый, читаемый, прокомментированный код; данные и палитра — сверху, легко заменить.
- Доступность: кликабельные элементы — с клавиатуры (focus-ring, `aria-label`), tooltip доступен.
- Без ошибок в консоли; деградирует корректно (нет WebGL → Canvas 2D fallback).

Верни готовый компонент/файл целиком.

## 🔧 Заметки для адаптации под InstaGuard (после генерации — делаю я)

- **Источник данных:** заменить `CAMPAIGN_STATS` на агрегат из БД. Новый роут (напр.
  `GET /api/stats/campaigns`) считает по владельцу сессии: группировка `TriggerRule` по `EventType`,
  сумма `fireCount` → главный срез; сумма `TriggerRule.stats` (dm/like/follow/story/comment) →
  срез «По действию»; группировка по `responder.username` → «По аккаунту».
- **Маппинг типов:** `EventType` → подпись/цвет уже описан таблицей выше (совпадает с UI проекта).
- **Стек:** Next.js 15 + Tailwind. Перенести стили на Tailwind + `lib/colors.ts` (`TONE`), шапку —
  на общий `PageHeader`, KPI — на `StatCard` (уже есть в проекте, редизайн 2026-07-04).
- **Тема/палитра:** бренд проекта фиолетовый `#663af1` (в `PROMPT` дал близкий `#7C5CFC` — свести
  к бренду при интеграции).
- **Reduced-motion / тач:** согласовать с решением по §13.12 (Tilt) — единый подход к анимациям.
- **Состояния:** добавить загрузку/пусто/ошибку (в проекте это обязательный чек — §14.2).
